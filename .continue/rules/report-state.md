# Analisis y depuracion de codigo por instruciones

1. epieza un analisis general del codigo
2. evalua test unitarios existentes,no los ejecutes solo verifica la
3. extructura del projecto
4. calidad del codigo 
5. ordena y priorisa el analisis hecho
6. genera una regla de las tareas faltantes
7. con el resultado obtenido de el analisis, genera tareas a realizar
8. segmentalas en etapas, decribiendo el problema
9. sugiere posibles alternativas segun el contexto de projecto, con objetivo de obtener la mejor calidad  
10. si no existe, crea el directorio docs dentro del workspace web
11. crea un archivo en markdonw
